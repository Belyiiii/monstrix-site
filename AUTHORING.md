# Как вести сайт MONSTRIX

Короткая инструкция: как править, как добавить проект, как открыть сбор.
Всё правится через **GitHub-приложение на телефоне** → репозиторий `monstrix-site` → нужный файл → правка → **Commit** в `main`. Через 1–2 минуты сайт пересобирается сам (GitHub Actions) и обновляется на monstrix.ru.

---

## Быстрый цикл

1. GitHub app → `monstrix-site` → открыть файл
2. Поправить текст (Markdown)
3. **Commit changes** → ветка `main`
4. Подождать 1–2 мин → готово

> Коммит в `main` = деплой. Больше ничего запускать не нужно.

---

## Где что лежит

| Путь | Что это |
|------|---------|
| `content/projects/<slug>.md` | **Один проект = один файл** |
| `content/firmware/` `guides/` `components/` | Прошивки, гайды, компоненты |
| `content/_index.md` | Интро-текст на главной (2 абзаца сверху) |
| `hugo.toml` | Общая донат-ссылка `donateURL` |
| `static/images/projects/<slug>/` | Картинки проекта |
| `layouts/`, `static/style.css`, `themes/` | Шаблоны и стили — **не трогать** (проси меня) |
| `public/` | Собирается автоматически — **не трогать** |

## Добавить новый проект

1. Создать файл `content/projects/<slug>.md`
   `<slug>` — латиница с дефисами, напр. `vfd-dashboard`.
2. Вставить шаблон ниже, заполнить поля.
3. Залить обложку в `static/images/projects/<slug>/cover.jpg` и прописать её в `cover`.
4. Commit → проект сам появится в разделе **Проекты** и на главной.

### Шаблон (копировать целиком в новый файл)

```yaml
---
title: "Название проекта"
date: 2026-07-12
cover: "/images/projects/vfd-dashboard/cover.jpg"
tags: ["esp32", "can"]

# --- MONSTRIX ---
stage: 1              # 0 = идея, 1 = в работе, 2 = готово
progress: 40          # только для stage 1 (0–100)
stuck_on: "текущий затык"   # пусто, если затыка нет
pinned: false         # true только у ОДНОГО проекта (закреп на главной)

goal_text: "Одно предложение: что и зачем"
term: "~2 месяца"
budget: "~8 000 ₽"

# Сбор со шкалой — заполнять ТОЛЬКО когда открыт сбор:
fund_url: ""          # ссылка Tribute на сбор; пусто = шкалы нет
fund_label: "Сбор на железо"
collected: 0          # руками, дублируешь сумму из Tribute
fund_goal: 8000

# Для готового проекта (stage 2):
github_url: ""
flash_url: ""         # напр. /firmware/vfd-dashboard/
kit_fund_url: ""      # сбор на партию
kit_count: ""         # напр. "3/10"
upgrade_fund_url: ""
upgrade_label: "Апгрейд: аккумулятор"
---

Тело статьи. Новые записи журнала дописываются СВЕРХУ.
```

Минимум для старта: `title`, `date`, `cover`, `tags`, `stage`, `goal_text`. Остальное — по мере надобности.

---

## Три стадии — что видит человек

| stage | Бейдж | Что показывается |
|:---:|---|---|
| 0 | `[ ИДЕЯ ]` | описание + что нужно от людей (`needs`) |
| 1 | `[ В РАБОТЕ ]` | прогресс-бар (`progress`) + затык (`stuck_on`) |
| 2 | `[ ГОТОВО ]` | блок «Забрать проект»: GitHub, прошивка, сбор на партию |

Общая **донат-кнопка** есть всегда; подпись меняется сама: незавершённый → «Сдвинуть проект», готовый → «Спасибо за проект».

## Обновить прогресс / затык

В файле проекта поменять:

- `progress: 70`
- `stuck_on: "расшифровка CAN пакетов"` (или пусто)

Commit — бар и строка затыка обновятся.

## Открыть СБОР со шкалой

Когда есть цель с конкретной суммой:

1. В **Tribute** создать «Сбор на цель», скопировать ссылку.
2. В файле проекта заполнить:
   - `fund_url: "<ссылка Tribute>"`
   - `fund_label: "Сбор на железо"` — подпись кнопки
   - `fund_goal: 8000`
   - `collected: 0` — обновляешь **руками** раз в 1–2 дня из Tribute
3. Commit → на карточке и странице появляется **вторая кнопка + шкала** `collected / fund_goal`.

**Закрыть сбор:** очистить `fund_url` (сделать `""`) — шкала пропадёт.

> Правило: **одна кнопка = «поддержать»** (общий донат), **две = идёт сбор**.

Партия («хочу такой же»): `kit_fund_url` + `kit_count: "3/10"`.
Апгрейд: `upgrade_fund_url` + `upgrade_label`.

---

## Деньги — два уровня

1. **Общий донат** — одна ссылка на весь сайт (`donateURL` в `hugo.toml`). Кнопка на каждом проекте; в сообщении к донату просят указать проект.
2. **Сбор со шкалой** — отдельная ссылка Tribute под конкретную цель (`fund_url`). Активных сборов держать 1–3, не больше.

---

## Журнал (дневник проекта)

Тело статьи под front matter. **Новая запись — всегда сверху:**

```markdown
## ── 12.07.2026 ──

![подпись](/images/projects/vfd-dashboard/foto.jpg)

Пара строк: что сделал, где застрял.
```

---

## Иконки — ТОЛЬКО ASCII, без эмодзи

Эмодзи на сайт **не добавляем**. Вместо них скобочные глифы:

| Смысл | Глиф | | Смысл | Глиф |
|---|---|---|---|---|
| ок / готово | `[✓]` | | питание | `[+-]` |
| нет / не так | `[x]` | | электроника | `[~]` |
| важно | `[!]` | | дисплей | `[#]` |
| вопрос | `[?]` | | сеть | `<>` |
| детали / файлы | `[+]` | | идея | `[i]` |
| цель / действие | `[>]` | | спасибо | `<3` |
| робот / эмоции | `[o_o]` | | стрелка (можно) | `→` |
